Proyecto Neptuno consists of 5 HTML pages, each with a specific purpose and semantic structure. All pages share a consistent layout pattern with header, main content, and footer.

## Common Structure

Every page follows this basic HTML5 structure:

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title — Proyecto Neptuno</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <header><!-- Navigation --></header>
    <main><!-- Page Content --></main>
    <footer><!-- Footer Links --></footer>
</body>
</html>
```

<Note>
  All pages use semantic HTML5 elements (`<header>`, `<main>`, `<footer>`, `<section>`, `<nav>`) for better accessibility and SEO.
</Note>

  <Tab title="contact.html">
    ### Contact Page
    
    Advanced form with CSS-only tab switching for different user profiles.
    
    #### Radio Tab System
    
    ```html
    <!-- Hidden radio inputs control tab state -->
    <input type="radio" id="p-inversor" name="perfil" value="inversor" class="radio-perfil">
    <input type="radio" id="p-marca" name="perfil" value="marca" class="radio-perfil">
    <input type="radio" id="p-prensa" name="perfil" value="prensa" class="radio-perfil">
    <input type="radio" id="p-ciudadano" name="perfil" value="ciudadano" class="radio-perfil">

    <!-- Tab labels -->
    <div class="profile-tabs">
        <label for="p-inversor" class="tab">Inversor</label>
        <label for="p-marca" class="tab">Marca</label>
        <label for="p-prensa" class="tab">Prensa</label>
        <label for="p-ciudadano" class="tab">Ciudadano</label>
    </div>
    ```
    
    #### Conditional Field Groups
    
    ```html
    <!-- Fields shown when p-inversor is checked -->
    <div id="fields-inversor" class="profile-fields">
        <div class="form-row form-row--two">
            <div class="form-group">
                <label for="inv-empresa">Empresa u organización</label>
                <input type="text" id="inv-empresa" name="inv_empresa">
            </div>
            <div class="form-group">
                <label for="inv-cargo">Cargo</label>
                <input type="text" id="inv-cargo" name="inv_cargo">
            </div>
        </div>
        <!-- More fields -->
    </div>
    ```
    
    **CSS Tab Logic:**
    
    ```css
    /* Hide all field groups by default */
    .profile-fields { display: none; }

    /* Show specific group when radio is checked */
    #p-inversor:checked ~ #fields-inversor { display: block; }
    #p-marca:checked ~ #fields-marca { display: block; }
    #p-prensa:checked ~ #fields-prensa { display: block; }
    #p-ciudadano:checked ~ #fields-ciudadano { display: block; }

    /* Style active tab */
    #p-inversor:checked ~ .profile-tabs label[for="p-inversor"] {
        color: var(--color-primario);
        border-bottom-color: var(--color-primario);
    }
    ```
    
    <Note type="info">
      The contact form uses **zero JavaScript** — all tab switching is handled with CSS sibling selectors (`~`) and the `:checked` pseudo-class.
    </Note>

## Semantic HTML Patterns

### Accessible Structure

All pages follow accessibility best practices:

<CodeGroup>
```html Header
<header>
    <div>
        <a href="./index.html"><img src="./img/logo.png" alt="Logo CC Neptuno"></a>
    </div>
    <nav>
        <ul>
            <li><a href="./index.html">Inicio</a></li>
            <li><a href="./ocio.html">Ocio</a></li>
            <li><a href="./gastro.html">Gastronomía</a></li>
            <li><a href="./proyecto.html">El Proyecto</a></li>
            <li><a href="./contact.html">Contacto</a></li>
        </ul>
    </nav>
</header>
```

```html Footer
<footer>
    <div class="footer-inner">
        <p class="footer-copy">2026 Proyecto Neptuno — Propuesta de renovación ficticia para Granada</p>
        <nav class="footer-nav" aria-label="Navegación secundaria">
            <a href="./index.html">Inicio</a>
            <a href="./ocio.html">Ocio</a>
            <a href="./gastro.html">Gastronomía</a>
            <a href="./proyecto.html">El proyecto</a>
            <a href="./contact.html">Contacto</a>
        </nav>
    </div>
</footer>
```
</CodeGroup>

**Accessibility Features:**
- Descriptive `alt` text on all images
- `aria-label` on secondary navigation
- `aria-hidden` on decorative elements
- Semantic heading hierarchy (h1 → h2)
- Form labels properly associated with inputs

## Responsive Considerations

All pages adapt to different screen sizes using CSS media queries:

- **Desktop (> 992px)**: Full multi-column layouts
- **Tablet (768px - 992px)**: Adjusted grids, hamburger menu
- **Mobile (< 576px)**: Single column, stacked layouts

<Note type="tip">
  The hamburger menu is implemented with pure CSS using a checkbox input and label system, similar to the contact form tabs.
</Note>
